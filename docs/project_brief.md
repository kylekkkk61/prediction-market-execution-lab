# Project Brief

## Working Title

**Prediction Market Execution Lab**

## Subtitle

**Testing Executable Edge in Polymarket BTC Short-Horizon Markets**

## Purpose

This project studies whether apparent pricing edges in short-horizon prediction markets can survive real execution frictions.

The public version is framed as a FinTech research and analytics project, not as a live trading or betting system.

## Core Research Question

Do short-horizon prediction-market prices contain apparent mispricings that remain meaningful after accounting for spread, slippage, fill probability, latency, position limits, and settlement outcomes?

## Initial Scope

- Market type: short-horizon BTC prediction markets
- Venue context: Polymarket-style prediction-market CLOB data
- Reference asset: BTC spot or perpetual reference prices
- Focus: executable edge, market microstructure, and execution quality

## Planned Deliverables

- Public README
- Methodology documentation
- Architecture documentation
- Limitations documentation
- Sample data schema
- Execution-quality report
- Probability-calibration report
- Reproducible notebooks
- Optional dashboard

## Public Positioning

This project is intended to demonstrate:

- probability modeling,
- market data engineering,
- microstructure-aware backtesting,
- execution-quality diagnostics,
- post-trade attribution,
- risk simulation,
- clear research communication.
